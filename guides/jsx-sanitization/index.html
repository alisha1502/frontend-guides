How JSX Protects You & How Babel Transpiles Your Code

When you write React code with JSX, two important things happen behind the scenes: JSX sanitization protects you from security vulnerabilities, and Babel transpilation converts your code into something browsers can understand.

üìä The Complete Flow

Step 1: You Write JSX
Your React components with JSX syntax
Step 2: Babel Transpilation
JSX ‚Üí JavaScript (happens at build time)
Step 3: React Runtime
React processes createElement calls
Step 4: JSX Sanitization
Values are escaped to prevent XSS attacks (happens at runtime)
Step 5: Browser Receives Safe HTML
Your app renders securely in the browser

üõ°Ô∏è Part 1: JSX Sanitization (Security)

üí° What is JSX Sanitization?

JSX automatically escapes (sanitizes) any values you put in curly braces {} to prevent XSS (Cross-Site Scripting) attacks. This means malicious code is displayed as text, not executed.

Example: How JSX Protects You

const UserProfile = () => { // Imagine a hacker entered this as their username: const username = '<img src=x onerror="alert(\'Hacked!\')" />'; return ( <div> <h1>Welcome, {username}!</h1> // ‚úÖ JSX sanitizes this! </div> ); }

What Gets Rendered:

Welcome, <img src=x onerror="alert('Hacked!')" />!

‚úÖ The malicious script is displayed as TEXT, not executed!

‚úÖ DO's and ‚ùå DON'Ts

‚úÖ DO: Let JSX Sanitize
<div> {userInput} </div> // ‚úÖ Safe - automatically escaped // Malicious code shown as text
SAFE
‚ùå DON'T: Bypass Sanitization
<div dangerouslySetInnerHTML={{ __html: userInput }} /> // ‚ùå DANGEROUS! // Executes malicious code
DANGEROUS

üîÑ Part 2: Babel Transpilation (Converting JSX)

üí° What is Babel Transpilation?

Browsers don't understand JSX syntax. Babel converts your JSX code into regular JavaScript using React.createElement() calls that browsers can execute.

Before & After Babel

üìù Before (JSX)
const Greeting = () => { const name = "John"; return ( <div className="container"> <h1>Hello, {name}!</h1> </div> ); }
Human-Friendly
‚öôÔ∏è After (JavaScript)
const Greeting = () => { const name = "John"; return React.createElement( "div", { className: "container" }, React.createElement( "h1", null, "Hello, ", name, "!" ) ); }
Browser-Ready

üìã Complete DO's and DON'Ts Reference

‚úÖ DO These Things
  • ‚úì Use {} for dynamic values
  • ‚úì Trust JSX's automatic sanitization
  • ‚úì Use expressions in curly braces
  • ‚úì Let Babel handle JSX conversion
  • ‚úì Display user input safely with {userInput}
‚ùå DON'T Do These
  • ‚úó Use dangerouslySetInnerHTML with user input
  • ‚úó Bypass JSX sanitization
  • ‚úó Try to manually transpile JSX
  • ‚úó Execute unescaped HTML from users
  • ‚úó Assume user input is safe

üéØ Key Takeaways

  • JSX automatically escapes values in {} to prevent XSS attacks
  • Babel transpiles JSX into React.createElement() calls
  • Sanitization happens at runtime, transpilation at build time
  • Never use dangerouslySetInnerHTML with untrusted data
  • You can use JavaScript expressions inside {}
  • Trust the process - JSX keeps your app secure by default

üöÄ Pro Tip

If you absolutely need to render HTML (e.g., from a trusted CMS), use a sanitization library like DOMPurify before passing it to dangerouslySetInnerHTML:

import DOMPurify from 'dompurify'; <div dangerouslySetInnerHTML={{ __html: DOMPurify.sanitize(trustedHTML) }} />
